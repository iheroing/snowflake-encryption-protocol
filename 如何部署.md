## 📝 两步部署

### 第一步：推送到 GitHub

#### 选项 A：使用部署脚本（推荐）

```bash
./deploy.sh
```

脚本会自动：
1. 检查远程仓库配置
2. 推送代码到 GitHub
3. 询问是否部署到 Vercel
4. 自动完成部署

#### 选项 B：手动操作

1. **在 GitHub 创建新仓库**
   - 访问 https://github.com/new
   - 仓库名: `snowflake-encryption-protocol`
   - 描述: `❄️ 雪花加密协议 - 基于分形算法的艺术加密应用`
   - 选择 Public
   - **不要**勾选 "Initialize with README"
   - 点击 "Create repository"

2. **推送代码**
   ```bash
   # 添加远程仓库（替换 YOUR_USERNAME）
   git remote add origin https://github.com/YOUR_USERNAME/snowflake-encryption-protocol.git
   
   # 推送代码
   git branch -M main
   git push -u origin main
   ```

---

### 第二步：部署到 Vercel

#### 选项 A：通过网站（最简单）

1. **访问 Vercel**
   - 打开 https://vercel.com
   - 点击 "Sign Up" 或 "Log In"
   - 选择 "Continue with GitHub"

2. **导入项目**
   - 点击 "Add New..." → "Project"
   - 在列表中找到 `snowflake-encryption-protocol`
   - 点击 "Import"

3. **配置（通常自动检测）**
   - Framework Preset: Vite ✅
   - Build Command: `npm run build` ✅
   - Output Directory: `dist` ✅
   - 直接点击 "Deploy"

4. **等待部署**
   - 大约 2-3 分钟
   - 完成后会显示链接

5. **访问你的网站**
   - 链接格式: `https://snowflake-encryption-protocol.vercel.app`
   - 或者: `https://snowflake-encryption-protocol-xxx.vercel.app`

#### 选项 B：通过 CLI

```bash
# 安装 Vercel CLI
npm install -g vercel

# 登录
vercel login

# 部署到生产环境
vercel --prod
```

## 🎨 自定义域名（可选）

1. 在 Vercel 项目中点击 "Settings"
2. 选择 "Domains"
3. 添加你的域名
4. 按照提示配置 DNS
5. 等待验证（通常几分钟）

## 🐛 遇到问题？

### GitHub 推送失败

```bash
# 检查远程仓库
git remote -v

# 如果错误，删除并重新添加
git remote remove origin
git remote add origin https://github.com/YOUR_USERNAME/snowflake-encryption-protocol.git
```

### Vercel 构建失败

1. 检查本地构建: `npm run build`
2. 查看 Vercel 构建日志
3. 确认 `package.json` 依赖正确

### 页面空白

1. 打开浏览器控制台
2. 查看错误信息
3. 检查网络请求
